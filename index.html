Run 0 — Core Principles

Architecture Rules

These eight rules are immutable. Every line of code in BCO Core must comply with them.

RULE 01
Single Source of Truth
One global state store in storage.js. No duplicate state sources.
RULE 02
Event-First Architecture
Everything originates as an event. State changes only after event processing.
RULE 03
Storage Abstraction
All storage through the adapter interface. Never call localStorage or DB directly.
RULE 04
Module Isolation
Modules communicate only via events and actions. No direct cross-module writes.
RULE 05
AI Non-Destructive
AI can suggest and analyse. It cannot mutate state directly or bypass governance.
RULE 06
Rule Engine Authority
Rules evaluated before every action. A blocking rule stops execution entirely.
RULE 07
UI Read-Only
UI dispatches events and renders state. It never mutates state directly.
RULE 08
No Hard Coupling
No direct storage calls outside the adapter. No hidden cross-layer dependencies.
